# Evidence Directory

Store test outputs, screenshots, and decision logs here.

## Naming convention

```
<issue-N>-<type>-<date>.<ext>
```

Examples:
- `1-test-output-2026-06-03.txt`
- `1-pack-dry-run-2026-06-03.txt`
- `2-review-verdict-2026-06-05.md`

## What to save

- `validate:quick` output (copy terminal output)
- `npm pack --dry-run` output
- Review gate verdict (PASS / PASS-WITH-NOTES / FAIL + notes)
- Architecture decision records for high-risk changes
